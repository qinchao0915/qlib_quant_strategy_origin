# 多模型协同选股方案

## 背景

当前策略是三个股票池独立运行：
- CSI300模型 → 只预测CSI300股票 → 选出Top N
- CSI500模型 → 只预测CSI500股票 → 选出Top N
- CSI1000模型 → 只预测CSI1000股票 → 选出Top N

**问题**：各跑各的，资金分散在三个独立策略中，无法全局优化配置。

## 目标

实现**跨股票池统一选股**：
```
CSI300模型 ─┐
CSI500模型 ─┼→ 同时预测全市场 → 合并排名 → 选出全局Top N
CSI1000模型─┘
```

## 方案设计

### 方案一：独立预测+归一化排名（推荐）

**核心思想**：每个模型预测自己擅长的股票池，然后用归一化方法统一排名

**步骤**：
1. **同时加载三个模型**
   ```python
   models_300, weights_300, features_300 = load_v7_model('csi300')
   models_500, weights_500, features_500 = load_v7_model('csi500')
   models_1000, weights_1000, features_1000 = load_v7_model('csi1000')
   ```

2. **分别获取三个股票池数据**
   ```python
   price_300 = fetcher.load_data('csi300')
   price_500 = fetcher.load_data('csi500')
   price_1000 = fetcher.load_data('csi1000')
   ```

3. **分别预测（各用各的模型）**
   ```python
   pred_300 = predict_ensemble(models_300, price_300)
   pred_500 = predict_ensemble(models_500, price_500)
   pred_1000 = predict_ensemble(models_1000, price_1000)
   ```

4. **Z-Score归一化**
   ```python
   # 因为三个模型输出范围不同，需要标准化
   pred_300_norm = (pred_300 - pred_300.mean()) / pred_300.std()
   pred_500_norm = (pred_500 - pred_500.mean()) / pred_500.std()
   pred_1000_norm = (pred_1000 - pred_1000.mean()) / pred_1000.std()
   ```

5. **合并并全局排序**
   ```python
   all_preds = pd.concat([pred_300_norm, pred_500_norm, pred_1000_norm])
   top_stocks = all_preds.nlargest(30)
   ```

6. **统一执行交易**
   - 不再区分股票池，统一风控参数
   - 记录每只股票来自哪个原始股票池（用于分析）

**优点**：
- 每个模型在自己熟悉的领域预测，准确性高
- 通过归一化实现跨池可比
- 可以灵活调整各池权重（如大盘行情加权CSI300）

**缺点**：
- 实现复杂度中等
- 需要处理好特征对齐问题

---

### 方案二：模型集成预测

**核心思想**：三个模型同时预测同一只股票，取平均或加权

**适用场景**：股票同时在多个指数中（实际上很少）

**不推荐**：因为CSI300/500/1000成分股几乎不重叠

---

### 方案三：分层配置

**核心思想**：资金按固定比例分配给三个股票池，各自独立运行

**配置示例**：
```python
ALLOCATION = {
    'csi300': 0.2,   # 20%资金给大盘
    'csi500': 0.5,   # 50%资金给中盘
    'csi1000': 0.3,  # 30%资金给小盘
}
```

**优点**：
- 实现简单
- 可以控制风格暴露

**缺点**：
- 不是真正的全局最优选股
- 可能出现某池没好票也硬买的情况

## 推荐实施方案：方案一（独立预测+归一化）

### 新增命令行参数
```bash
python3 main.py --combined --start 2025-01-01 --end 2025-12-31 --mode backtest
```

### 数据结构变更

**交易记录新增字段**：
```python
{
    '代码': '000001',
    '名称': '平安银行',
    '所属池': 'csi300',  # 新增：标记来自哪个原始股票池
    '原始得分': 0.045,   # 新增：模型原始预测值
    '归一得分': 2.35,   # 新增：Z-Score后得分
    '全局排名': 5,       # 新增：在所有股票中的排名
    # ... 原有字段
}
```

### 实现步骤

1. **修改`load_v7_model`**：支持一次性加载多个模型
2. **新增`CombinedDataFetcher`**：同时获取三个股票池数据
3. **新增`CombinedPredictor`**：分别预测+归一化+合并
4. **修改`run_backtest`**：支持传入合并后的预测结果
5. **修改交易记录**：添加来源股票池标记

### 风控调整

统一风控参数（不再按股票池区分）：
```python
COMBINED_CONFIG = {
    'max_positions': 30,
    'max_per_stock': 0.1,  # 单票10%
    'max_per_industry': 0.3,  # 行业30%
    'hard_stop': -0.08,
    'trailing_stop': 0.12,  # 统一12%
}
```

### 预期效果

- **更优的资金配置**：资金自动流向当前最有价值的股票，不受股票池限制
- **更好的风险分散**：可以在三个池之间动态平衡
- **更准确的信号**：比如CSI300整体都很差时，可以全部不买或只买其中最优秀的

### 对比测试建议

1. **独立策略组合**：CSI300(33%) + CSI500(34%) + CSI1000(33%)
2. **协同选股策略**：全局Top 30，不限定股票池比例

对比指标：
- 收益率
- 夏普比率
- 最大回撤
- 换手率
- 行业集中度
